Executive summary

摘要：我们要造什么？

  • 推理基础软件：在终端提供开源模型的统一加载、执行与观测层，屏蔽 CPU/GPU/NPU 差异，沉淀内存与线程策略。
  • 类小程序容器：第三方交付轻量包（UI + 逻辑 + manifest），通过稳定 Bridge 调用模型能力与受限系统能力，支持热更新与权限审计。
  • 异构调度：同一设备上共存 LLM、扩散、VLM 与传统小模型，按会话、优先级与热/电量状态做队列与降级，避免「大模型一跑，全场卡顿」。
  • 生态接口：模型以「能力」注册（schema、SLA、许可证），应用以依赖声明消费，便于评测、合规与版本兼容矩阵。

价值闭环

运行时负责「算力与内存」，Bridge 负责「稳定 ABI」，小程序专注交互与编排。

Problem space

背景与动机：为何需要这层软件？

端侧趋势

隐私合规、弱网离线、交互延迟敏感；开源权重与端侧引擎可用性快速提升，但工程集成高度碎片化。

开发者成本

每个 App 重复实现加载/量化路径、内存预算、前后处理与回退策略；跨模态 API 不一致导致业务代码难以复用。

生态缺口

缺少类似小程序的标准化沙箱与分发机制，模型能力难以像服务一样被第三方安全组合。

研究价值

统一调度框架下可量化研究异构共存、动态路由、能耗–质量 Pareto，而非一次性 Demo。

碎片化示意 · 多应用 × 多模型组合爆炸

缺少共享运行时 → 重复打包、重复优化、行为不一致。

  • 结论：需要「操作系统式」的端侧 AI 基座：把共性沉到底层，把创新留给上层小程序。
  • 边界：不追求单一模型 SOTA；强调可插拔、可观测、可治理与多模型协同。
Positioning

愿景与利益相关方

愿景：成为移动端开源智能应用的默认运行时底座——开发者主要编写业务与交互，平台处理算力、并发与合规钩子。

终端用户

默认本地推理与显式外传同意；可预期延迟与功耗；多应用间体验一致。

应用开发者

统一 Bridge 与调试探针；专注 manifest 与 UI；减少底层推理排障时间。

模型社区

标准包与基准入口；许可证与能力元数据可读；促进复用而非 Fork 碎片化。

平台 / 系统

算力预算、热管理、权限审计与遥测（可匿名）；为 OEM 集成预留策略接口。

非目标：不替代完整移动 OS；初期不承诺覆盖全部芯片后端，以适配器插件演进。

Architecture

逻辑分层：稳定 API，可插拔后端

L5 智能小程序 / 第三方 SDK · UI · manifest
Bridge API · 稳定 ABI / IDL
L4 应用框架：权限 · 生命周期 · 包管理 · 沙箱
L3 能力编排：多模态管线 · 会话 · 工具调用适配
L2 推理调度：队列 · 优先级 · 批处理 · 降级策略
L1 模型运行时：LLM · Diffusion · VLM · 传统小模型
L0 硬件抽象：CPU / GPU / NPU · 内存池 · 量化编译

设计原则

  • 向上稳定：Bridge 长期兼容；内部实现可换。
  • 策略可插拔：调度与降级以策略包形式下发。
  • 观测优先：延迟/显存/功耗 trace 一等公民。
Model families

LLM 与扩散：两类「重任务」如何共处？

大语言模型

  • 自回归解码对显存与带宽敏感；利用前缀 KV 复用与会话级公平。
  • 交互式请求可打断后台摘要；支持投机解码若后端允许。
  • 端侧以中小参数 + 强量化为主；超大模型走可配置云端回退与隐私脱敏。

图示：交互式负载占比（示例）

扩散模型

  • 多步迭代：暴露步数自适应与提前终止，映射质量–时间曲线。
  • 与 LLM 错峰调度，避免与 VLM 同事抢占显存峰值。
  • 产品层：进度、取消、缩略预览，保证可感知响应。
Multimodal & tiny ML

VLM 与传统小模型：感知与门控

视觉-语言模型

  • 视觉编码器往往决定首包延迟：提供分辨率档位与「先小图后大图」策略。
  • 与相机/相册 IO 并行：预读、降采样与缓存帧。
  • 隐私：本地默认；上传需显式声明与最小必要原则。
相机帧→降采样→VLM→结构化输出

专用小模型

  • 低延迟判别：检测、关键词、唤醒、轻量 ASR 等。
  • 作为路由门控：先判场景/意图，再决定是否唤醒大模型。
  • 调度上通常标记为高优先级短任务，避免被生成式任务饿死。
Scheduler

统一调度：把「并发」变成可策略化的问题

  • 全局预算：随温度、电量与插电状态收紧并发与 batch。
  • 任务分类：交互式 / 后台 / 生成式（可取消）——不同 SLA。
  • 跨模型 DAG：检测 → 裁剪 → VLM → LLM 总结；节点级超时与替代路径。
  • 降级路径：备用小模型、降分辨率/步数/上下文；可解释地反馈给 UI。

示例流水线

调度器在 DAG 边上加超时、重试与替代算子，避免单点拖垮整条链路。

Developer experience

「类小程序」开发者模型

  • 交付物：manifest（模型依赖、最低算力档、权限声明）+ 前端资源 + 业务脚本。
  • 推理 API：以能力 ID 调用，输入输出符合注册 schema；支持流式与取消令牌。
  • 系统 API：存储、受控网络、相机等——默认最小授权，宿主可策略收紧。
  • 嵌入形态：宿主 App 集成 SDK；后期可选独立启动器聚合多小程序。

Trust

安全、隐私与供应链

  • 沙箱：文件/网络/敏感 API 默认关闭；宿主策略可统一审计。
  • 数据驻留：默认本地推理；任何上传需二次确认与用途说明。
  • 模型供应链：签名、哈希、SBOM；构建可重现与漏洞响应流程。
  • 内容安全：生成式输出策略（年龄模式、企业关键词）由宿主配置。

Roadmap

技术路线：验证 → 平台 → 生态

单平台（建议 Android）；接入 LLM + 小模型；打通 Bridge 与示例小程序；发布模型包格式 v0.1 与基线评测。

Diffusion / VLM 适配器；多队列抢占与更细粒度遥测；开发者工具链（模拟器、剖析、合规扫描）与 RFC 流程。

模型注册表与可选应用分发形态；跨 App 互操作规范；iOS / 策略限制单独评估。

Open & metrics

开源治理与可量化成功标准

  • 许可证：Runtime 采用 Apache-2.0/MIT 等宽松协议；模型权重遵循上游。
  • 治理：RFC 重大变更；SemVer；安全公告渠道。
  • 学术产出：调度策略、异构共存、端侧 QoS、能耗–质量权衡等可发表系统性结果。
